# Agile and DevOps

Agile and DevOps are complementary philosophies that, when used together, bridge the gap between ideation and production. While Agile focuses on managing the development process through iterative "sprints" and customer feedback, DevOps extends this culture into the operations side, using automation to ensure that software is delivered and maintained reliably.

## How They Work Together

The most efficient software teams use Agile to figure out what to build and DevOps to handle how to ship it.

Agile Interactions + DevOps Pipelines: Customer discussions provide the work, while DevOps CI/CD pipelines provide the engine that automatically tests and deploys code as it is completed.

Continuous Feedback: Agile provides feedback from users on functionality; DevOps provides feedback from production on system performance and stability.

Shared Ownership: In a combined model, the distinction between "development" and "operations" fades. Everyone is responsible for the software's success from the first line of code to its live operation.
