# Putting the cart before the horse

Question: In a technology project context, what does "putting the cart before the horse" most commonly refer to?

- [ ] Using agile methodology instead of waterfall
- [ ] Implementing features before establishing clear goals and a strategic plan
- [ ] Documenting code after it has been written
- [ ] Testing software before deploying to production

<details>
  <summary>Answer</summary>
  <p>Implementing features before establishing clear goals and a strategic plan</p>
  <p>The idiom "putting the cart before the horse" means doing things in the wrong order or prioritizing incorrectly. In technology projects, this typically manifests as jumping into implementation details or specific tasks before establishing clear goals, objectives, or a strategic plan. The phrase emphasizes the importance of proper planning and following a logical sequence of steps to achieve success efficiently.</p>
</details>
