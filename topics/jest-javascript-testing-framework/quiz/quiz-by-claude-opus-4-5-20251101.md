# Jest JavaScript testing framework

Question: What is a primary strength of Jest that allows developers to start testing immediately without extensive setup?

- [ ] Requires integration with multiple third-party assertion libraries
- [ ] Zero-configuration approach with built-in test runners, assertion libraries, and mocking capabilities
- [ ] Mandatory manual configuration of test file locations
- [ ] Exclusive support for React applications only

<details>
  <summary>Answer</summary>
  <p>Zero-configuration approach with built-in test runners, assertion libraries, and mocking capabilities</p>
  <p>Jest's zero-configuration approach is one of its primary strengths, allowing developers to start testing immediately without extensive setup. It comes with built-in test runners, assertion libraries, mocking capabilities, and code coverage reporting, eliminating the need to configure multiple testing tools.</p>
</details>
