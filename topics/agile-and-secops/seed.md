# Agile and secops

In modern development, SecOps (Security Operations) combined with Agile is the practice of "shifting left"—moving security from a final gatekeeper role at the end of a project into the heart of every sprint. 

## How SecOps Integrates into Agile

Rather than waiting for a separate security audit, security becomes a shared responsibility that evolves alongside the code. 

Security User Stories: Agile teams include specific security requirements (e.g., "API must require OAuth2 tokens") as Acceptance Criteria in their backlog. 

Topic-Based Threat Modelling: During topic planning, teams perform lightweight threat modelling to identify risks in new features before a single line of code is written. 

Automated Guardrails: Security tools (like SAST and DAST) are built directly into the CI/CD pipeline to catch vulnerabilities automatically with every code commit.
