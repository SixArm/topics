# Test tooling

Question: Which test automation tool uses a behavior-driven development (BDD) syntax with Given-When-Then steps that allows non-technical stakeholders to understand test scenarios?

- [ ] Jest
- [ ] Selenium
- [ ] Cucumber
- [ ] Playwright

<details>
  <summary>Answer</summary>
  <p>Cucumber</p>
  <p>Cucumber is a test automation runner that uses Gherkin syntax, which expresses test scenarios in plain language using Given-When-Then steps. This BDD approach enables collaboration between developers, testers, and business stakeholders by making test cases readable and understandable without requiring programming knowledge.</p>
</details>
