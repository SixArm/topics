# Agile chartering

Question: What is the primary purpose of agile chartering in project management?

- [ ] To create detailed technical specifications and architecture documents
- [ ] To assign individual tasks and deadlines to each team member
- [ ] To collaboratively define the vision, goals, and objectives while aligning all stakeholders toward a shared direction
- [ ] To establish a fixed budget and timeline that cannot be changed

<details>
  <summary>Answer</summary>
  <p>To collaboratively define the vision, goals, and objectives while aligning all stakeholders toward a shared direction</p>
  <p>Agile chartering brings together diverse stakeholders—including team members, customers, end-users, and business owners—to collaboratively establish the project vision, set SMART objectives, identify risks, and create a working agreement. The core purpose is ensuring all stakeholders are aligned and working toward a shared goal, which creates a framework for success and enables more effective teamwork.</p>
</details>
