# Out of Scope

Question: What is the primary purpose of declaring something "out of scope" in project management?

- [ ] To permanently reject client requests and end business relationships
- [ ] To delay work indefinitely without providing alternatives
- [ ] To indicate that a task or activity is not within the bounds of the current project or assignment
- [ ] To avoid accountability for project deliverables

<details>
  <summary>Answer</summary>
  <p>To indicate that a task or activity is not within the bounds of the current project or assignment</p>
  <p>Declaring something "out of scope" establishes clear boundaries for what is included in the current project. This doesn't mean the request is rejected permanentlyâ€”it means it cannot be addressed within the existing project parameters. Clients or stakeholders can initiate a new project or change request to address out-of-scope items. This practice helps manage expectations, prevent scope creep, and maintain project timelines and budgets.</p>
</details>
