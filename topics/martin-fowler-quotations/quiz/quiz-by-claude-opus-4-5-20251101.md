# Martin Fowler quotations

Question: According to Martin Fowler, what is the defining characteristic of good programmers compared to any fool who can write code?

- [ ] Good programmers write code that executes faster than average
- [ ] Good programmers write code that humans can understand
- [ ] Good programmers write code with the fewest lines possible
- [ ] Good programmers write code that never needs refactoring

<details>
  <summary>Answer</summary>
  <p>Good programmers write code that humans can understand</p>
  <p>Fowler's famous quotation "Any fool can write code that a computer can understand. Good programmers write code that humans can understand" emphasizes that clear, expressive code helps future developers and reduces the cognitive burden of understanding complex systems. This principle underlies many software engineering best practices including meaningful naming, proper documentation, and thoughtful code organization.</p>
</details>
