# Premature optimization is the root of all evil

Question: According to the principle "premature optimization is the root of all evil," what is the recommended approach to improving software performance?

- [ ] Optimize all code sections equally from the start of development
- [ ] Sacrifice code readability early to ensure maximum performance
- [ ] Profile the code to identify actual bottlenecks before optimizing
- [ ] Apply micro-optimizations throughout the codebase as you write it

<details>
  <summary>Answer</summary>
  <p>Profile the code to identify actual bottlenecks before optimizing</p>
  <p>Donald Knuth's principle advises against premature optimization because it can lead to wasted effort, increased complexity, and unnecessary trade-offs. Instead, developers should use a data-driven approach: profile the code, measure actual performance, and focus optimization efforts on the areas that genuinely need improvement rather than optimizing speculatively.</p>
</details>
