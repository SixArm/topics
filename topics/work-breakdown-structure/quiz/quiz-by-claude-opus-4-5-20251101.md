# Work Breakdown Structure (WBS)

Question: What is the primary purpose of a Work Breakdown Structure (WBS) in project management?

- [ ] To assign budgets to individual team members based on their roles
- [ ] To hierarchically decompose a project into smaller, manageable components for organizing scope
- [ ] To create a timeline showing task dependencies and critical path
- [ ] To document stakeholder communication preferences and contact information

<details>
  <summary>Answer</summary>
  <p>To hierarchically decompose a project into smaller, manageable components for organizing scope</p>
  <p>A WBS is a hierarchical decomposition of a project into smaller, more manageable components such as tasks, activities, or packages. It provides a structured framework for organizing and understanding the scope of work required for a project, which enables scope control, task identification, estimation, scheduling, and stakeholder communication.</p>
</details>
