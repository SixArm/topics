# Ralph loop

The Ralph loop is an autonomous AI coding technique where an agent—often using tools like Claude Code or Amp—is put into a continuous, self-iterating feedback loop until a task is completed. It repeatedly feeds its own errors and output back into itself until the code works.How It WorksTask Automation: You feed the AI a set of goals or a Product Requirements Document (PRD), and it breaks the project down into specific, atomic tasks.Continuous Iteration: The agent tackles one task per iteration. When it hits a wall or tries to exit, a stop hook intercepts it, reinjects the original prompt, and starts a fresh instance with the updated context.State Persistence: While each iteration runs in a clean session, progress is retained through Git history and tracking files (like progress.txt or prd.json) so the AI remembers what has already been tried.Origins & Use CasesThe Name: It is named after Ralph Wiggum from The Simpsons, representing a stubborn, persistent approach of "try, try again" until it works.Best Uses: It is highly effective for automating tedious, mechanical work like library upgrades, refactoring, and version bumps, rather than for creative architectural planning.Best Practices: Developers often use it with stacked PRs (Pull Requests) to make the agent's output easier to review, and they set maximum iteration limits to prevent infinite token burning.To learn more about the mechanics, the origins of the pattern, or to read real developer reviews, check out Geoffrey Huntley's Ralph Pattern Article
