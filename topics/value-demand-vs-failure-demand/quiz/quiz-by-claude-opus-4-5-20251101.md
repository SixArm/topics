# Value demand vs failure demand

Question: What is the key distinction between value demand and failure demand in agile software engineering?

- [ ] Value demand is work requested by management, while failure demand is work requested by customers
- [ ] Value demand refers to high-priority tasks, while failure demand refers to low-priority tasks
- [ ] Value demand directly contributes to customer satisfaction and business objectives, while failure demand is work generated by system failures or deficiencies
- [ ] Value demand is planned work in sprints, while failure demand is unplanned work outside sprints

<details>
  <summary>Answer</summary>
  <p>Value demand directly contributes to customer satisfaction and business objectives, while failure demand is work generated by system failures or deficiencies</p>
  <p>Value demand represents work that creates tangible benefits users recognizeâ€”such as new features, enhanced user experience, and requested functionality. Failure demand encompasses work caused by failures like bug fixes, rework, technical debt, and resolving issues from system deficiencies. Understanding this distinction helps teams optimize resource allocation by maximizing value-creating activities and minimizing work that consumes resources without adding new value.</p>
