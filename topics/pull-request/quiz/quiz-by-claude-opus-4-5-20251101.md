# Pull request

Question: What is the primary purpose of a pull request (PR) in Git?

- [ ] To automatically deploy code changes to production servers
- [ ] To create a backup copy of the repository on a remote server
- [ ] To allow developers to propose, review, and discuss changes before merging them into the main branch
- [ ] To delete obsolete branches from the repository

<details>
  <summary>Answer</summary>
  <p>To allow developers to propose, review, and discuss changes before merging them into the main branch</p>
  <p>A pull request serves as a collaborative mechanism where developers can propose changes to a codebase, enabling other team members to review the code, provide feedback, and discuss modifications before the changes are merged into the main branch. This process improves code quality, facilitates collaboration, and maintains a clear change log.</p>
</details>
