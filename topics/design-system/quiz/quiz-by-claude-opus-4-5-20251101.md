# Design system

Question: What is the primary purpose of a design system in digital product development?

- [ ] To replace the need for user testing and feedback
- [ ] To maintain consistency and cohesion across user interfaces through shared guidelines, principles, and reusable components
- [ ] To eliminate the need for developers to write any custom code
- [ ] To create unique, one-off designs for each screen in an application

<details>
  <summary>Answer</summary>
  <p>To maintain consistency and cohesion across user interfaces through shared guidelines, principles, and reusable components</p>
  <p>A design system serves as a central resource that defines visual and interactive elements, patterns, and rules to be followed when creating user interfaces. It includes design principles, visual style guides, component libraries, interaction patterns, accessibility guidelines, and documentation. This improves UI/UX consistency, efficiency, scalability, and collaboration across teams.</p>
</details>
