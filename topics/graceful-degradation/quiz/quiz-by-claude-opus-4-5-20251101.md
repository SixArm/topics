# Graceful degradation

Question: What is the primary purpose of graceful degradation in UI/UX design?

- [ ] To ensure websites only work on the latest browsers with cutting-edge features
- [ ] To maintain usability and core functionality when certain features or technologies are not supported in a user's environment
- [ ] To completely disable a website when JavaScript is turned off
- [ ] To prioritize visual aesthetics over functional accessibility

<details>
  <summary>Answer</summary>
  <p>To maintain usability and core functionality when certain features or technologies are not supported in a user's environment</p>
  <p>Graceful degradation ensures that a website or application remains functional and provides a positive user experience even when certain features or technologies are unavailable. This approach prioritizes core functionality so all users can access and interact with essential features regardless of their device, browser, or technological limitations.</p>
</details>
