# Unified Medical Language System (UMLS)

The Unified Medical Language System (UMLS) is a comprehensive set of files and software tools developed by the [U.S. National Library of Medicine (NLM)](https://www.nlm.nih.gov/research/umls/index.html) to facilitate [interoperability between disparate biomedical information systems](https://pmc.ncbi.nlm.nih.gov/articles/PMC6115234/). [1, 2, 3] 
It acts as a "bridge" that brings together hundreds of different health and biomedical vocabularies—such as RxNorm, SNOMED CT, ICD-10, and LOINC—into a single, unified framework. [4, 5, 6, 7] 

## The Three Knowledge Sources

The UMLS is built around three core components: [8] 

* Metathesaurus: The central database containing over 1 million biomedical concepts and 5 million names. It links synonymous terms from different vocabularies to a single Concept Unique Identifier (CUI). [7, 9, 10] 
* Semantic Network: A system that categorizes every concept in the Metathesaurus into broad "types" (e.g., Disease or Syndrome, Medical Device) and defines the relationships between them. [9, 11] 
* SPECIALIST Lexicon & Lexical Tools: A set of resources designed for Natural Language Processing (NLP). It helps computers handle variations in spelling, suffixes, and word forms in medical text. [4, 6] 

## Why It Is Used

* Electronic Health Records (EHRs): Allows patient data to be shared accurately between hospitals, pharmacies, and insurance providers. [5] 
* Research & Data Mining: Enables researchers to pull data from multiple sources (like [MEDLINE](https://www.nlm.nih.gov/medline/medline_home.html) and clinical databases) using a single set of search terms. [1] 
* Translation: Facilitates the mapping of terms between different systems, such as converting a doctor's clinical notes into billing codes. [8]
