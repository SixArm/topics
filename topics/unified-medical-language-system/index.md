# Unified Medical Language System (UMLS)

The Unified Medical Language System (UMLS) is a comprehensive set of files and software tools developed by the U.S. National Library of Medicine (NLM) to facilitate interoperability between disparate biomedical information systems. It acts as a bridge that brings together hundreds of different health and biomedical vocabularies, such as RxNorm, SNOMED CT, ICD-10, and LOINC, into a single unified framework.

The UMLS is built around three core components. The Metathesaurus is the central database containing over one million biomedical concepts and five million concept names. It links synonymous terms from different vocabularies to a single Concept Unique Identifier (CUI), so that "heart attack," "myocardial infarction," and their equivalents across coding systems all resolve to one canonical concept. The Semantic Network categorizes every concept in the Metathesaurus into broad types, such as "Disease or Syndrome" or "Medical Device," and defines the relationships between them. The SPECIALIST Lexicon and Lexical Tools provide resources designed for natural language processing (NLP), helping software handle variations in spelling, suffixes, and word forms found in medical text.

In practice, UMLS supports several critical functions. For electronic health records (EHRs), it allows patient data to be shared accurately between hospitals, pharmacies, and insurance providers that use different coding systems internally. For research and data mining, it enables investigators to query multiple sources, such as MEDLINE and clinical databases, using a single set of search terms rather than translating queries manually. For clinical-to-administrative translation, it facilitates mapping between systems, such as converting a clinician's notes into standardized billing codes. These capabilities make UMLS a foundational layer for any system that needs to process, exchange, or reason over biomedical terminology at scale.
