# Reactive UI

Question: What is the primary characteristic that distinguishes declarative UI in reactive frameworks from traditional UI development approaches?

- [ ] It requires manual DOM manipulation to update visual elements
- [ ] It focuses on imperatively specifying step-by-step UI updates
- [ ] It describes how the UI should look based on the current data state
- [ ] It eliminates the need for any data binding mechanisms

<details>
  <summary>Answer</summary>
  <p>It describes how the UI should look based on the current data state</p>
  <p>Declarative UI is a fundamental concept in reactive frameworks where developers describe what the UI should look like given the current state of the data, rather than imperatively manipulating the UI step by step. This approach allows the framework to automatically determine how to update the UI when the underlying data changes, resulting in cleaner, more maintainable code and reducing the risk of UI state becoming out of sync with the data model.</p>
</details>
