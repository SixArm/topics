# Acceptance testing

Question: What is the primary goal of acceptance testing in software development?

- [ ] To identify memory leaks and optimize code performance
- [ ] To verify that the software meets the client's requirements and performs as expected
- [ ] To test individual units of code in isolation
- [ ] To find bugs in the source code before integration

<details>
  <summary>Answer</summary>
  <p>To verify that the software meets the client's requirements and performs as expected</p>
  <p>Acceptance testing evaluates whether a software application meets the requirements and specifications of the client or user. It is conducted to ensure the software is ready for deployment and use by end-users, typically performed by end-users, business analysts, or quality assurance professionals who assess functionality, usability, and performance.</p>
</details>
