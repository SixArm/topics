# Functional specifications - steps

Question: What is the primary purpose of defining acceptance criteria in a functional specification?

- [ ] To categorize requirements for easier understanding
- [ ] To establish specific, measurable conditions that can be tested to determine if the system is acceptable
- [ ] To identify features requested by stakeholders
- [ ] To describe how users interact with the system

<details>
  <summary>Answer</summary>
  <p>To establish specific, measurable conditions that can be tested to determine if the system is acceptable</p>
  <p>Acceptance criteria define the conditions the software must meet to be considered acceptable. They must be specific, measurable, and testable, serving as the benchmark for validating that the system meets its requirements before delivery.</p>
</details>
