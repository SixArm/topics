# Agile and specification-driven development (SDD)

Agile and specification-driven development (SDD) combines structured upfront requirements with iterative, AI-assisted coding to prevent AI hallucinations and costly errors. It treats specifications as contracts—concise, evolving documents—that define scope, constraints, and testable acceptance criteria, which AI agents then implement, ensuring high-quality, maintainable code. 

## Core Principles of Spec-Driven Development

AI as Developer: Specifications are created for AI agents to follow,, reducing guesswork and unauthorized deviations. 

Living Documentation: Unlike traditional, rigid specifications, these documents are continually updated in Markdown by the team as development proceeds. 

Structure: Work is divided into small, manageable tasks, often following frameworks like openspec, kitab, or spec kit. 

Validation: Specifications clearly state how the work will be verified, often using Behavioral Driven Development (BDD) techniques like "Given/When/Then". 

## Workflow in AI-Assisted SDD

Research: The AI reviews the current codebase to understand context.

Plan: A detailed plan, outlining constraints and goals, is written and reviewed by a human.

Execute: The AI agent uses the spec to generate functional code, improving, or "fixing," wandering agents. 

## Agile vs. Spec-Driven Debate

While some consider this a return to "Waterfall," proponents argue that it is actually a modern evolution of Agile, allowing for faster iteration while providing the necessary structure for AI to work accurately. It ensures the AI isn't just generating code quickly, but generating the correct code.
