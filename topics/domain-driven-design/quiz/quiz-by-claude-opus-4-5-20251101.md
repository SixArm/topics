# Domain-Driven Design (DDD)

Question: What is the primary purpose of "Ubiquitous Language" in Domain-Driven Design?

- [ ] To define the programming language used for implementation
- [ ] To create technical documentation for developers only
- [ ] To establish a shared vocabulary between business stakeholders and the development team
- [ ] To standardize API naming conventions across microservices

<details>
  <summary>Answer</summary>
  <p>To establish a shared vocabulary between business stakeholders and the development team</p>
  <p>Ubiquitous Language is a core practice in DDD that ensures both business stakeholders and developers use the same terminology when discussing the domain. This shared language reduces misunderstandings, improves communication, and helps ensure the software accurately reflects business requirements and goals.</p>
</details>
