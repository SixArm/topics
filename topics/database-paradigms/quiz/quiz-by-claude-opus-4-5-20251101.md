# Database paradigms

Question: Which database paradigm is best suited for managing complex, interconnected data such as social networks and recommendation systems?

- [ ] Document-oriented database
- [ ] Relational database
- [ ] Graph database
- [ ] Object-oriented database

<details>
  <summary>Answer</summary>
  <p>Graph database</p>
  <p>Graph databases store data as nodes and edges, where nodes represent entities and edges represent relationships between them. This structure makes them ideal for managing highly interconnected data like social networks, where relationships between users are central, and recommendation systems, which rely on traversing complex relationship paths efficiently.</p>
</details>
