# Use cases

Question: What is the primary purpose of a use case in software engineering?

- [ ] To document the technical architecture and database schema of a system
- [ ] To describe and define the interactions between a user or system and a product
- [ ] To estimate project timelines and resource allocation for development teams
- [ ] To generate automated test scripts for continuous integration pipelines

<details>
  <summary>Answer</summary>
  <p>To describe and define the interactions between a user or system and a product</p>
  <p>A use case is a technique used to capture functional requirements by outlining the steps taken to achieve a specific goal, identifying inputs, outputs, and actors involved. This helps ensure the product meets user needs and provides a foundation for testing and validating functionality.</p>
</details>
