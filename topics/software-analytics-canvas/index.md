# Softare analytics canvas

The Software Analytics Canvas is a structured framework created by Markus Harrer that guides technology teams through data-driven analysis of their software systems, processes, and organizations. It provides a repeatable, step-by-step approach to ensure that each analysis effort is well-scoped, methodical, and produces actionable outcomes.

The canvas walks practitioners through seven steps. It begins with defining a clear question: what specifically do you want to know, who cares about the answer, and what would change if you had the result. Without a well-formed question, analysis efforts tend to drift toward symptoms rather than root causes.

Next, you identify relevant data sources and assess whether they can actually answer your question. This includes evaluating what information is available, what is missing, and whether manual data collection is needed. Choosing the right data upfront prevents false conclusions downstream.

The heuristics step requires you to state your assumptions and simplifications explicitly. Real-world data rarely answers a question precisely, so you document the approximations you are making. The validation step then defines what a useful result looks like and how it will be presented to decision-makers in a comprehensible way.

Implementation planning comes before writing any analysis code. Sketching your approach and tooling choices first keeps the analysis reproducible and understandable. Overly complex analysis code undermines trust in the results.

The final steps capture results and next steps. You document key insights, including negative or inconclusive findings, and derive concrete follow-up actions. An analysis that does not lead to decisions or experiments is incomplete.

A side benefit of completing the canvas is that it produces documentation of the entire investigation, making it shareable and reusable for future analyses.
